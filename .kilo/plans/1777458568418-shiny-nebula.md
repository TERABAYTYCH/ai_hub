# План реализации приложения Control

## Обзор

Реализация приложения `control` по аналогии с `service` и `pulse` — микросервис для управления командами устройств с NestJS backend (BFF паттерн) и React/Vite frontend (Module Federation).

**Архитектурный паттерн:**
- Backend: BFF (Backend-for-Frontend) — лёгкий NestJS с проксированием запросов в Hub
- Frontend: Module Federation remote, загружается в Hub

## Этапы реализации

### 1. Backend (`apps/control/backend`)

**Структура:**
```
apps/control/backend/
├── src/
│   ├── main.ts
│   ├── app.module.ts
│   ├── app.controller.ts
│   └── hub-proxy/
│       ├── hub-proxy.module.ts
│       ├── hub-proxy.controller.ts
│       └── index.ts
├── tsconfig.json
├── Dockerfile
└── package.json
```

**Файлы:**

1. **package.json** — скопировать с service-backend, изменив имя на `@ject-hub/control-backend`

2. **tsconfig.json** — скопировать с service-backend

3. **Dockerfile** — скопировать с service-backend, заменив `3002` на `3003` и `service-backend` на `control-backend`

4. **main.ts** — CORS с `http://control.lvh.me` в списке allowedOrigins, порт `3003`

5. **app.module.ts** — ConfigModule + HubProxyModule + AppController

6. **app.controller.ts** — эндпоинт `/control/health`

7. **hub-proxy/** — BFF для проксирования запросов к Hub (аналогично service-backend)

### 3. Docker Compose

Добавить сервисы:

```yaml
control-backend:
  build:
    context: .
    dockerfile: apps/control/backend/Dockerfile
  container_name: ${COMPOSE_PROJECT_NAME}_control_backend
  restart: unless-stopped
  environment:
    - NODE_ENV=development
    - PORT=3003
    - JWT_SECRET=${JWT_SECRET:-hub-jwt-secret-key-change-in-production}
    - HUB_INTERNAL_API_URL=http://hub-backend:3000
  ports:
    - '3003:3003'
  volumes:
    - .:/app
    - /app/node_modules
    - /app/apps/control/backend/node_modules
  command: yarn workspace @ject-hub/control-backend start:dev
  depends_on:
    - hub-backend
  networks:
    - ject-network

control-frontend:
  build:
    context: .
    dockerfile: apps/control/frontend/Dockerfile
  container_name: ${COMPOSE_PROJECT_NAME}_control_frontend
  restart: unless-stopped
  environment:
    - NODE_ENV=development
    - VITE_API_URL=http://api.hub.lvh.me
    - WATCHPACK_POLLING=true
  ports:
    - '5176:5176'
  volumes:
    - .:/app
    - /app/node_modules
    - /app/apps/control/frontend/node_modules
    - ./apps/control/frontend/dist:/app/apps/control/frontend/dist
  command: yarn workspace @ject-hub/control-frontend dev:docker
  depends_on:
    - control-backend
  networks:
    - ject-network
```

**Важно:** Добавить `control-backend` и `control-frontend` в `depends_on` nginx и volumes для apps/control/frontend/dist в nginx volume.

### 4. Nginx (⚠️ ПРОБЛЕМА)

**Проблема:** `nginx/nginx.conf` помечен как **ЗАПРЕЩЕНО ИЗМЕНЯТЬ**.

Нужно добавить маршрутизацию:
- `control.lvh.me` → `control-frontend:5176`
- `api.control.lvh.me` → `control-backend:3003`

**Варианты решения:**
1. Спросить разрешение на изменение nginx.conf
2. Или добавить маршруты в существующую инфраструктуру через отдельный конфиг

---

### 5. Hub Frontend Integration

Hub frontend динамически загружает модули из pulse и service. Для полной интеграции Control необходимо:

1. Добавить `control.lvh.me` в `allowedHosts` vite.config.ts hub-frontend
2. Добавить `control` в manifests (через Module Federation manifest plugin)

---

## Volumes

Добавить в docker-compose.yml volumes секцию nginx:
```yaml
volumes:
  - ./apps/control/frontend/dist:/usr/share/nginx/html/control:ro
```
