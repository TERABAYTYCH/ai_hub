# План: Вынесение Layout в ui-kit и универсальная навигация

## Цель

Вынести Layout из Hub в ui-kit, чтобы Pulse и Service могли отображать Hub Settings и другие сервисы в сайдбаре.

## Задачи

### Тикет 006: Pulse — Добавить Service и Hub в навигацию

**Текущее состояние:**
- `ui-kit/Layout.tsx` — уже есть, использует manifests
- `REGISTERED_SERVICES` — pulse и service, НЕТ hub
- Hub vite.config.ts — нет exposes и manifestPlugin

**Меню для Pulse после изменений:**

```
- Dashboard (/pulse/dashboard)
- Devices (/pulse/devices)
- Metrics (/pulse/metrics)
- Alerts (/pulse/alerts)
- Settings (/pulse/settings)
- Service (submenu)
  - Dashboard → /service/dashboard
  - Devices → /service/devices
  - Settings → /service/settings
- Hub Settings → /hub/settings
```

---

#### 006.1: Hub → Federation host + manifest

**Изменить `apps/hub/frontend/vite.config.ts`:**

```typescript
federation({
  name: 'hub',
  filename: 'remoteEntry.js',
  exposes: {
    './Settings': './src/pages/SettingsPage',
  },
  shared: ['react', 'react-dom', 'react-router-dom'],
})
```

+ добавить manifestPlugin:
```typescript
manifestPlugin({
  serviceId: 'hub',
  serviceName: 'Ject Hub',
  moduleMapping: {
    './Settings': { label: 'Settings', icon: 'bi bi-gear', path: '/hub/settings' },
  },
})
```

**Проверки:**
- [ ] Hub отдаёт `http://hub.lvh.me/assets/remoteEntry.js` с ./Settings
- [ ] Hub отдаёт `http://hub.lvh.me/assets/manifest.json`

---

**Проверки:**
- [ ] Все 3 сервиса в списке

2. `apps/service/frontend/vite.config.ts` — использовать из libs/plugins

**Проверки:**
