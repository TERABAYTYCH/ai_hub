# План: PERMISSIONS_001 — Управление доступом к микросервисам через JWT

## Цель

Добавить в JWT информацию о доступности микросервисов (pulse, service). На фронтенде:

- Заблокированный сервис отображается с замком (disabled) в навигации
- При прямом входе на заблокированный сервис — "Модуль недоступен"

## Архитектура

### JWT Payload

```typescript
{
  sub: "user-id",
  username: "...",
  role: "admin|user",
  licenseId: "...",
  microservices: {
    pulse: true,   // доступен
    service: false // заблокирован
  }
}
```

### UI для заблокированного сервиса

**В навигации:** 🔒 Service (disabled)

**При входе на заблокированный сервис:**

```
┌─────────────────────────┐
│  🔒 Модуль недоступен    │
│  Для активации Service  │
│  обратитесь к администратору│
└─────────────────────────┘
```

### URL

- `/hub/lock` — LockPage для Hub
- `/pulse/lock` — LockPage для Pulse
- `/service/lock` — LockPage для Service

#### 017.2: Hub Backend — MicroservicesController API

**Контекст:** Для управления доступом к микросервисам нужен API endpoint. Этот контроллер позволит администратору включать/выключать доступ к Pulse и Service в реальном времени. Пока данные хранятся в памяти (в будущем — в БД).

**Создать `apps/hub/backend/src/microservices/microservices.controller.ts`:**

```typescript
@Controller('microservices')
export class MicroservicesController {
  @Get('access')
  getAccess() {
    return MICROSERVICES_ACCESS;
  }

  @Post('access')
  updateAccess(@Body() body: { pulse?: boolean; service?: boolean }) {
    if (body.pulse !== undefined) {
      MICROSERVICES_ACCESS.pulse = body.pulse;
    }
    if (body.service !== undefined) {
      MICROSERVICES_ACCESS.service = body.service;
    }
    return MICROSERVICES_ACCESS;
  }
}
```

**Создать модуль `apps/hub/backend/src/microservices/microservices.module.ts`:**

```typescript
@Module({
  controllers: [MicroservicesController],
})
export class MicroservicesModule {}
```

**Импортировать в AppModule (apps/hub/backend/src/app.module.ts).**

**Проверки:**

- [ ] `GET /api/microservices/access` возвращает `{ pulse: true, service: false }`
- [ ] `POST /api/microservices/access` с `{ "service": true }` обновляет настройки

#### 017.4: Hub Frontend — Проверка доступа в App.tsx

**Контекст:** Сейчас `useDynamicRoutesConfig()` в `apps/hub/frontend/src/App.tsx` загружает модули из всех сервисов без проверки доступа. Нужно добавить проверку: если `microservicesAccess[serviceId] === false`, редиректить на `/{serviceId}/lock`.

**Где искать:** `apps/hub/frontend/src/App.tsx` — функция `useDynamicRoutesConfig()` (строки 102-144)

**Что делать:**
В `useDynamicRoutesConfig` добавить проверку `isLocked`:

```typescript
const useDynamicRoutesConfig = () => {
  const { user } = useAuth();
  const { manifests } = useMicroserviceManifests();
  const microservicesAccess = user?.microservices || {};

  const routes = manifests.flatMap(manifest => {
    const isLocked = microservicesAccess[manifest.serviceId] === false;

    if (isLocked) {
      return [{
        path: `/${manifest.serviceId}`,
        element: <Navigate to={`/${manifest.serviceId}/lock`} replace />,
      }];
    }

    return manifest.navigation.map(nav => ({
      path: nav.path,
      element: <LazyModule serviceId={manifest.serviceId} modulePath={nav.module} />,
    }));
  });

  return routes;
};
```

**Проверки:**

- [ ] При `microservices.service === false` и попытке зайти на `/service` происходит редирект на `/service/lock`

#### 017.6: Hub Frontend — MicroservicesSettings

**Контекст:** Администратору нужна страница для управления доступом к модулям. Она будет обращаться к `POST /api/microservices/access` для сохранения настроек.

**Создать `apps/hub/frontend/src/pages/MicroservicesSettings.tsx`:**

```typescript
export default function MicroservicesSettings() {
  const [access, setAccess] = useState({ pulse: true, service: true });
  const [saved, setSaved] = useState(false);

  useEffect(() => {
    axios.get('/api/microservices/access').then(res => setAccess(res.data));
  }, []);

  const handleToggle = (service: 'pulse' | 'service') => {
    setAccess(prev => ({ ...prev, [service]: !prev[service] }));
    setSaved(false);
  };

  const handleSave = async () => {
    await axios.post('/api/microservices/access', access);
    setSaved(true);
    setTimeout(() => setSaved(false), 2000);
  };

  return (
    <div className="container mt-4">
      <h2>Управление доступом к модулям</h2>
      <Form.Check
        type="switch"
        id="pulse-switch"
        label="Pulse"
        checked={access.pulse}
        onChange={() => handleToggle('pulse')}
      />
      <Form.Check
        type="switch"
        id="service-switch"
        label="Service"
        checked={access.service}
        onChange={() => handleToggle('service')}
      />
      <button className="btn btn-primary mt-3" onClick={handleSave}>Применить</button>
      {saved && <span className="ms-3 text-success">Сохранено!</span>}
    </div>
  );
}
```

**Добавить route в App.tsx:**

```typescript
{ path: '/microservices-settings', element: <MicroservicesSettings /> }
```

**Проверки:**

- [ ] Страница `/microservices-settings` работает
- [ ] Toggle переключает состояние
- [ ] Кнопка "Применить" сохраняет настройки

#### 016.2: Infrastructure — Sidebar locked items

**Контекст:** Sidebar (в `libs/ui-kit/src/components/layout/Sidebar.tsx`) отображает навигационное меню. Нужно добавить поддержку `locked` элементов.

**Важно:** Заблокированный сервис НЕ имеет детей (это обрабатывается в Layout 016.3). Sidebar получает items, где locked items уже без children.

**Где искать:** `libs/ui-kit/src/components/layout/Sidebar.tsx` — интерфейс `MenuItem` (строка 8-19)

**Добавить в `MenuItem`:**

```typescript
export interface MenuItem {
  id: string;
  label: string;
  icon?: string;
  path?: string;
  children?: MenuItem[];
  locked?: boolean; // <-- добавить
}
```

**Изменить `renderMenuItem`:**
Если `item.locked === true`:

- Использовать `bi bi-lock` вместо `item.icon`
- НЕ показывать шеврон ( chevron)
- При клике — навигация на `item.path` (этот path ведёт на /lock)

```typescript
// Внутри renderMenuItem, для родительского элемента с детьми:
if (hasChildren) {
  // НЕ показывать шеврон если locked
  {!item.locked && (
    <i className={`bi bi-chevron-${isExpanded ? 'up' : 'down'} ms-2`} style={{ fontSize: '0.75rem' }}></i>
  )}
}

// Дочерний элемент (Nav.Link) для locked item:
if (item.locked && item.path) {
  return (
    <Nav.Link
      as={Link}
      to={item.path}
      className={`...`}
      style={{ opacity: 0.6 }}
    >
      <i className={`bi bi-lock me-2`}></i>
      {item.label}
    </Nav.Link>
  );
}
```

- [ ] Sidebar рендерит locked items с иконкой замка

#### 016.3: Infrastructure — Layout microservicesAccess

**Контекст:** UiKit Layout (в `libs/ui-kit/src/components/layout/Layout.tsx`) строит динамическое меню из manifest.json. Нужно добавить проп `microservicesAccess` и правильно обрабатывать заблокированные сервисы.

**Где искать:** `libs/ui-kit/src/components/layout/Layout.tsx`

**Добавить проп:**

```typescript
interface LayoutProps {
  children: ReactNode;
  serviceName?: string;
  staticMenuItems?: MenuItem[];
  excludeServices?: string[];
  hubSettingsItem?: MenuItem;
  microservicesAccess?: Record<string, boolean>; // <-- добавить
}
```

**Логика для ЗАБЛОКИРОВАННОГО сервиса (microservicesAccess[serviceId] === false):**

- ОДИН пункт меню (parent), БЕЗ детей
- Иконка: `bi bi-lock`
- НЕТ шевлеона (chevron)
- path: `/{serviceId}` (весь клик на этот пункт редиректит на lock)
- При клике — редирект на `/{serviceId}/lock`

**Логика для ДОСТУПНОГО сервиса (microservicesAccess[serviceId] !== false):**

- Parent с children (как раньше)
- Иконка: `bi bi-grid`
- Есть шеврон
- Дети — как обычно

**Изменить логику построения `serviceMenuItems`:**

```typescript
const serviceMenuItems: MenuItem[] = filteredManifests.map((m) => {
  const isLocked = microservicesAccess?.[m.serviceId] === false;

  // Заблокированный сервис — без детей, с замком, клик ведёт на /lock
  if (isLocked) {
    return {
      id: m.serviceId,
      label: m.name,
      icon: 'bi bi-lock',
      locked: true,
      path: `/${m.serviceId}`,
    };
  }

  // Доступный сервис — с детьми и шевроном
  return {
    id: m.serviceId,
    label: m.name,
    icon: 'bi bi-grid',
    children: m.navigation.map((n) => ({
      ...n,
    })),
  };
});
```
