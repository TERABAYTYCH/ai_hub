# ADR-001: Инициализация базовой инфраструктуры монорепозитория

## Статус: Принято

## Контекст

Проект Ject Hub требует настройки с нуля. Необходимо создать монорепозиторий с поддержкой множественных микросервисов, общих библиотек и изолированной инфраструктуры для локальной разработки.

## Решение

### 1. Yarn Workspaces + Turborepo

- Корневой `package.json` настроен с workspaces для `apps/*/*` и `libs/*`
- `turbo.json` определяет задачи: `build`, `dev`, `lint`, `typecheck`, `test`
- Используется Yarn v1.22.22 как пакетный менеджер

### 2. Структура директорий

Создана полная структура для 4 сервисов (Hub, Pulse, Service, Control), каждый с frontend и backend, плюс 3 общие библиотеки (contracts, ui-kit, core-backend).

### 3. Docker Compose инфраструктура

**Базы данных (MySQL 8.0):**
- 4 изолированных MySQL контейнера: `mysql-hub`, `mysql-pulse`, `mysql-service`, `mysql-control`
- Порты: 3306, 3307, 3308, 3309 соответственно
- Каждая БД создает свой database и user через ENV переменные
- Healthcheck для мониторинга готовности

**Kafka (KRaft mode):**
- Apache Kafka 3.7.0 в режиме KRaft (без ZooKeeper)
- Экономия ресурсов и упрощение инфраструктуры
- Порт 9092 для брокера

**Nginx (API Gateway):**
- Базовая конфигурация с health check endpoint
- Подготовлены заглушки для маршрутизации API запросов
- Rate limiting zone для будущей защиты

### 4. Сетевая изоляция

Все сервисы находятся в общей Docker сети `ject-network` для внутренней коммуникации через DNS имена.

## Последствия

### Положительные
- Полностью изолированная среда для разработки
- Автоматическое создание баз данных без SQL скриптов
- Возможность независимой разработки каждого сервиса
- Healthcheck обеспечивают мониторинг готовности сервисов

### Отрицательные
- 4 отдельных MySQL контейнера потребляют больше ресурсов
- При необходимости можно объединить в один контейнер с разными БД

## Альтернативы

Рассматривался вариант с одним MySQL контейнером и созданием 4 баз данных внутри него, но был отклонен в пользу полной изоляции на начальном этапе.
