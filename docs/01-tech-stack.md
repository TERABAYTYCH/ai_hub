# Технологический стек и правила написания кода

## 1. Основной стек

- **Язык:** TypeScript (Strict mode включен везде).
- **Backend:** NestJS (Express под капотом).
- **Frontend:** React + Vite (с использованием Module Federation).
- **Базы данных:** MySQL + TypeORM.
- **Брокер сообщений:** Kafka (или RabbitMQ, согласно текущему ADR).
- **Пакетный менеджер:** Yarn (используются Yarn Workspaces / Turborepo). ЗАПРЕЩЕНО использовать npm или pnpm.
- **Инфраструктура:** Docker и Docker Compose (обязательно для локальной разработки и деплоя).

## 2. Жесткие правила типизации и линтера

- **ЗАПРЕЩЕНО** использовать тип `any`. Если тип неизвестен, используй `unknown` или опиши интерфейс.
- **ЗАПРЕЩЕНО** игнорировать ошибки TypeScript (`@ts-ignore` допускается только в крайних случаях с обязательным комментарием `// FIXME:`).
- Все общие интерфейсы (DTO, контракты Kafka) должны импортироваться ТОЛЬКО из `libs/contracts`. Запрещено дублировать интерфейсы внутри микросервисов.
- Код должен соответствовать правилам ESLint и Prettier, настроенным в корне проекта.

## 3. Архитектурные и инфраструктурные запреты

- **Изоляция БД:** Микросервисы не имеют права ходить в чужие базы данных напрямую. Только через API или асинхронные события.
- **Docker Compose:** Вся локальная разработка идет через контейнеры. Фронтенд и бэкенд контейнеры должны поддерживать hot-reload (через проброс volumes).
- **Бизнес-логика:** Контроллеры NestJS должны быть тонкими. Вся бизнес-логика пишется в `*.service.ts`.
- **Фронтенд:** Никаких прямых запросов к чужим бэкендам в обход API Gateway (Nginx).
