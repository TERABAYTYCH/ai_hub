# Task 002: Реорганизация структуры управления задачами

## 🎯 Цель

Привести папку `docs/tasks/` к доменной структуре. Создать подпапки для микросервисов, перенести туда существующие задачи и изменить формат именования отчетов на `.report.md` для правильной алфавитной сортировки (чтобы файл задачи `.md` всегда был выше файла отчета `.report.md`).

## 🛑 Ограничения (CRITICAL)

1. **Только файловые операции:** Не изменяй содержимое самих файлов задач.
2. **Колокация:** Файлы отчетов (`.report.md`) должны лежать строго в тех же папках, что и файлы самих задач. Папка `reports/` больше не используется.

---

## 📋 Шаги выполнения (Step-by-Step)

- [ ] **Шаг 1: Создание структуры директорий**
  - Перейди в директорию `docs/tasks/`.
  - Создай папки: `000-infrastructure/`, `100-hub/`, `200-pulse/`, `300-service/`, `400-control/`.
  - Добавь пустой файл `.gitkeep` в папки `100-hub`, `200-pulse`, `300-service` и `400-control`.

- [ ] **Шаг 2: Перемещение и переименование**
  - Найди в корне `docs/tasks/` файлы инфраструктурных задач (`001-init.md`, `001-fixup-configs.md`, `002-reorganize-tasks.md`) и перемести их в `docs/tasks/000-infrastructure/`.
  - Найди все существующие файлы отчетов (например, `001-init.done.md`, `001-fixup-configs.done.md`). Перемести их в `docs/tasks/000-infrastructure/` и **переименуй**, заменив `.done.md` (или любой другой суффикс) на `.report.md` (например, `001-init.report.md`).
  - Удали папку `docs/tasks/reports/`, если она пуста.

- [ ] **Шаг 3: Обновление документации состояния**
  - Открой файл `docs/04-current-state.md`.
  - Добавь запись о том, что структура задач реорганизована, а отчеты теперь хранятся рядом с задачами с суффиксом `.report.md` для корректной сортировки в IDE.

- [ ] **Шаг 4: Отчет и завершение**
  - Создай файл-отчет `docs/tasks/000-infrastructure/002-reorganize-tasks.report.md`.
  - В отчете выведи получившееся дерево файлов директории `docs/tasks/`.
  - Предложи сообщение для коммита (например, `chore(docs): reorganize tasks and rename reports to .report.md`).
