# Task 002: Реорганизация структуры управления задачами

## 🎯 Цель

Привести папку `docs/tasks/` к доменной структуре для удобного управления монорепозиторием. Необходимо создать подпапки для каждого микросервиса и инфраструктуры, а затем перенести туда существующие файлы задач вместе с их отчетами (`.done.md`).

## 🛑 Ограничения (CRITICAL)

1. **Только перемещение:** Не изменяй содержимое самих файлов задач (`001-init.md`, `001-fixup-configs.md` и т.д.). Твоя задача — только работа с файловой системой.
2. **Колокация:** Файлы отчетов (`.done.md`) должны лежать строго в тех же папках, что и файлы самих задач. Папка `reports/` больше не используется.

---

## 📋 Шаги выполнения (Step-by-Step)

- [ ] **Шаг 1: Создание структуры директорий**
  - Перейди в директорию `docs/tasks/`.
  - Создай следующие папки:
    - `000-infrastructure/` (для задач по Docker, CI/CD, корневым конфигам)
    - `100-hub/` (для задач ядра: frontend + backend)
    - `200-pulse/` (для сервиса мониторинга)
    - `300-service/` (для сервиса обслуживания)
    - `400-control/` (для сервиса управления)
  - Добавь пустой файл `.gitkeep` в папки `100-hub`, `200-pulse`, `300-service` и `400-control`.

- [ ] **Шаг 2: Перемещение существующих задач и отчетов**
  - Найди в корне `docs/tasks/` файлы инфраструктурных задач (например, `001-init.md`, `001-fixup-configs.md`, `002-reorganize-tasks.md`).
  - Перемести их в папку `docs/tasks/000-infrastructure/`.
  - Найди все существующие файлы отчетов (например, `001-init.done.md`, `001-fixup-configs.done.md`) и также перемести их в `docs/tasks/000-infrastructure/`, чтобы они лежали рядом со своими задачами.
  - Удали папку `docs/tasks/reports/`, если она существует и пуста.

- [ ] **Шаг 3: Обновление документации состояния**
  - Открой файл `docs/04-current-state.md`.
  - Добавь запись о том, что структура задач была реорганизована по доменам, а отчеты теперь хранятся рядом с задачами (колокация).

- [ ] **Шаг 4: Отчет и завершение**
  - Создай файл-отчет `docs/tasks/000-infrastructure/002-reorganize-tasks.done.md`.
  - В отчете выведи получившееся дерево файлов директории `docs/tasks/`.
  - Предложи сообщение для коммита (например, `chore(docs): reorganize tasks structure and apply colocation for reports`).
