# Task 000-infrastructure/005: Настройка Docker Dev-окружения (Hot-Reload)

## 🎯 Цель

Обновить существующий `docker-compose.yml` для запуска проекта в режиме разработки с поддержкой Hot-Reload (горячей перезагрузки), чтобы избежать пересборки образов при каждом изменении кода.

## 🧭 Контекст

В данный момент при любых изменениях в коде мы вынуждены пересобирать Docker-образы целиком. Это занимает много времени. Нам нужно настроить dev-окружение прямо в основном `docker-compose.yml`, пробросив исходный код с хост-машины внутрь контейнеров через `volumes`, и заменить команды запуска на dev-скрипты (с флагом watch).

## 📋 Задачи для выполнения

### 1. Обновление `docker-compose.yml`
- Для сервиса `hub-backend`:
  - Добавить монтирование корневой директории монорепозитория в рабочую директорию контейнера (например, `- ./:/usr/src/app` — сверьтесь с `WORKDIR` в Dockerfile).
  - Добавить анонимный том для `node_modules`, чтобы модули внутри контейнера не затирались хостовыми: `- /usr/src/app/node_modules`.
  - Изменить команду запуска (`command`) на скрипт разработки: `yarn workspace @ject-hub/hub-backend start:dev` (или эквивалентный).
- Для сервиса `hub-frontend`:
  - Добавить аналогичное монтирование корня проекта и анонимный том для `node_modules`.
  - Изменить команду запуска (`command`) на: `yarn workspace @ject-hub/hub-frontend start` (или `dev`).
  - Убедиться, что в `environment` фронтенда прокинуты нужные переменные (например, `WATCHPACK_POLLING=true`, если Hot-Reload в Docker работает нестабильно).

### 2. Проверка работы
- Перезапустить контейнеры (`docker-compose down` и `docker-compose up --build`).
- Внести тестовое изменение в код бэкенда (например, добавить `console.log` в `main.ts`) и убедиться, что NestJS автоматически перезагрузился в логах контейнера.
- Внести тестовое изменение в код фронтенда и убедиться, что изменения мгновенно отразились в браузере.

## 🏁 Критерии приемки (DoD)
1. `docker-compose.yml` обновлен: добавлены `volumes` и dev-команды.
2. При изменении кода в `apps/hub/backend` или `apps/hub/frontend` сервисы автоматически перезагружаются внутри контейнеров без пересборки образа.
3. Написан отчет в формате `005-docker-dev-environment.report.[YYYYMMDD-HHMM].md`.
