# Task 000-infrastructure/003: Внедрение React Bootstrap и системы темизации (Dark/Light)

## 🎯 Цель

Интегрировать `react-bootstrap` во все фронтенд-приложения монорепозитория. Создать переиспользуемую логику переключения светлой/темной темы (Dark/Light mode) в общей библиотеке `libs/ui-kit` и применить её в сервисе Hub.

## 🧭 Контекст

Для ускорения разработки пользовательских интерфейсов во всех микросервисах (Hub, Pulse, Service, Control) принято решение использовать React Bootstrap. Кроме того, современный UX требует наличия темной темы "из коробки". Bootstrap 5.3+ поддерживает нативную темизацию через атрибут `data-bs-theme`. Нам нужно создать общий провайдер темы и компонент-переключатель, чтобы не дублировать этот код в каждом из 4-х фронтендов.

## 📋 Задачи для выполнения

### 1. Установка зависимостей
- Добавить пакеты `bootstrap` и `react-bootstrap` в рабочие пространства `libs/ui-kit` и `apps/hub/frontend` (используя `yarn workspace ... add ...`).
- Убедиться, что зависимости корректно резолвятся в монорепозитории.

### 2. Разработка в `libs/ui-kit`
- Создать контекст `ThemeProvider` (React Context), который будет:
  - Управлять состоянием текущей темы (`light` или `dark`).
  - Читать начальное значение из `localStorage` (или системных настроек `window.matchMedia('(prefers-color-scheme: dark)')`).
  - Сохранять выбранную тему в `localStorage`.
  - Устанавливать атрибут `data-bs-theme="dark|light"` на тег `<html>` или `<body>` при монтировании и изменении темы.
- Создать компонент `ThemeToggle` (кнопка на базе React Bootstrap), который потребляет `ThemeProvider` и переключает тему по клику. Иконки для кнопки можно реализовать текстом (☀️/🌙) или через легковесные SVG.
- Экспортировать `ThemeProvider` и `ThemeToggle` из главного файла `libs/ui-kit/src/index.ts`.

### 3. Интеграция во фронтенд Hub (`apps/hub/frontend`)
- Импортировать глобальные стили Bootstrap в корневой файл приложения (`main.tsx` или `App.tsx`): 
  `import 'bootstrap/dist/css/bootstrap.min.css';`
- Обернуть корневой компонент приложения в `ThemeProvider` из `@ject-hub/ui-kit` (или как назван пакет в `package.json`).
- Добавить компонент `ThemeToggle` в навигационную панель (Navbar) или шапку приложения.
- Переписать существующую тестовую страницу Hub с использованием компонентов `react-bootstrap` (например, `Container`, `Row`, `Col`, `Card`, `Button`), чтобы продемонстрировать работу стилей.

## 🛠 Требования и ограничения
- **Strict Mode:** TypeScript в строгом режиме, без `any`.
- **Экспорт UI Kit:** Убедиться, что `libs/ui-kit` корректно собирается через Turborepo (если требуется сборка) или корректно транспайлится Vite-плагинами из фронтенда.
- **FOUC (Flash of Unstyled Content):** Постараться минимизировать мерцание темы при загрузке страницы (насколько это возможно в SPA).

## 🏁 Критерии приемки (DoD)
1. Зависимости `bootstrap` и `react-bootstrap` установлены.
2. В `libs/ui-kit` созданы и экспортированы `ThemeProvider` и `ThemeToggle`.
3. Фронтенд Hub успешно запускается (`yarn turbo run dev --filter=hub-frontend`).
4. На странице Hub отображается кнопка переключения темы.
5. При клике на кнопку тема меняется со светлой на темную и обратно (меняются цвета компонентов Bootstrap).
6. Выбранная тема сохраняется после перезагрузки страницы (работает `localStorage`).
7. Написан и сохранен отчет `003-shared-ui-bootstrap.report.md` в папке `docs/tasks/000-infrastructure/`.
