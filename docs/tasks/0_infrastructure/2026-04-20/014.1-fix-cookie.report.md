# Task 014.1: Исправление атрибута Domain для кросс-доменных кук

## Проблема

SSO между `hub.localhost` и `pulse.localhost` не работал:
- `ject_access_token` получал домен `.hub.localhost` вместо `.localhost`
- `ject_refresh_token` вообще не имел атрибута Domain, поэтому привязывался к `api.hub.localhost`

## Решение

### Шаг 1: Frontend getRootDomain() - Уже исправлен

В предыдущей итерации `getRootDomain()` уже был исправлен:

```typescript
export function getRootDomain(): string | undefined {
  const hostname = window.location.hostname;

  if (hostname === 'localhost' || /^[0-9.]+$/.test(hostname)) {
    return undefined;
  }

  if (hostname.endsWith('.localhost')) {
    return '.localhost';
  }

  const parts = hostname.split('.');
  if (parts.length >= 2) {
    return `.${parts.slice(-2).join('.')}`;
  }

  return undefined;
}
```

### Шаг 2: Backend Set-Cookie - Исправлен

Добавлена функция `computeCookieDomain()` и обновлены все endpoints:

```typescript
function computeCookieDomain(host: string): string {
  const hostname = host.split(':')[0];

  if (hostname === 'localhost' || /^[0-9.]+$/.test(hostname)) {
    return '';
  }

  if (hostname.endsWith('.localhost')) {
    return 'Domain=.localhost; ';
  }

  const parts = hostname.split('.');
  if (parts.length >= 2) {
    return `Domain=.${parts.slice(-2).join('.')}; `;
  }

  return '';
}
```

**Примеры Set-Cookie:**
- Login: `ject_refresh_token=xxx; Domain=.localhost; Path=/; HttpOnly; SameSite=Lax; Max-Age=604800`
- Logout: `ject_access_token=; Domain=.localhost; Path=/; Expires=Thu, 01 Jan 1970 00:00:00 GMT`
