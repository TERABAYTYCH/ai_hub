# Fix Task: Исправление атрибута Domain для кросс-доменных кук

SSO между `hub.localhost` и `pulse.localhost` всё ещё не работает. Анализ DevTools показал, что куки устанавливаются на неверные домены:

- `ject_access_token` получает домен `.hub.localhost` (вместо `.localhost`).
- `ject_refresh_token` вообще не имеет атрибута Domain в заголовке `Set-Cookie`, из-за чего привязывается к `api.hub.localhost`.

Твоя задача — исправить логику выдачи доменов.

---

### Шаг 1. Исправление getRootDomain() на фронтенде

Текущая реализация возвращает текущий поддомен вместо корневого.

1. Открой `libs/ui-kit/src/utils/cookies.ts`.
2. Перепиши `getRootDomain()` так, чтобы она **гарантированно** отрезала поддомены. Если хост `hub.localhost` или `pulse.localhost`, она ДОЛЖНА вернуть `.localhost`.

```typescript
export function getRootDomain(): string | undefined {
  const hostname = window.location.hostname;
  if (hostname === 'localhost' || /^[0-9.]+$/.test(hostname)) return undefined;

  // Если это локальная разработка с поддоменами
  if (hostname.endsWith('.localhost')) return '.localhost';

  // Для продакшена (отрезаем всё, кроме последних двух частей)
  const parts = hostname.split('.');
  if (parts.length >= 2) {
    return `.${parts.slice(-2).join('.')}`;
  }
  return undefined;
}
```

---

### Шаг 2. Добавление Domain в Set-Cookie на бэкенде

Бэкенд сейчас вообще не передает домен для Refresh-токена.

1. Открой `apps/hub/backend/src/auth/auth.controller.ts`.
2. В методах `login` и `refresh` бэкенд должен вычислять корневой домен (на основе заголовка `Host` или `Origin` из `req`) и добавлять его в строку.

```typescript
// Пример вычисления на бэкенде:
const host = req.headers.host || '';
let cookieDomain = '';

if (host.includes('.localhost')) {
  cookieDomain = 'Domain=.localhost; ';
} else if (host.includes('.')) {
  const parts = host.split(':')[0].split('.');
  if (parts.length >= 2) {
    cookieDomain = `Domain=.${parts.slice(-2).join('.')}; `;
  }
}

const refreshTokenCookie = `ject_refresh_token=${loginResponse.refreshToken}; ${cookieDomain}Path=/; HttpOnly; SameSite=Lax; Max-Age=604800`;
```

3. Сделай то же самое для метода `logout` — при удалении кук тоже нужно указывать правильный `Domain`, иначе браузер их не удалит!

### 🛑 Самопроверка (Strict Check)

Перед завершением задачи убедись:

- [ ] **Фронтенд:** `getRootDomain()` для `hub.localhost` возвращает ровно `.localhost`.
- [ ] **Бэкенд:** В строке `Set-Cookie` для `ject_refresh_token` явно присутствует `Domain=.localhost;` (или другой вычисленный корневой домен).
- [ ] **Бэкенд (Logout):** В методе `logout` куки очищаются с указанием того же самого `Domain`.
