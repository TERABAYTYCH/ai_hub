# Task 001.1: Доработка корневой инфраструктуры (Ignore-файлы, Makefile, Jest, TS)

## 🎯 Цель

Дополнить базовую инициализацию монорепозитория. Необходимо создать файлы игнорирования (`.gitignore`, `.dockerignore`), `Makefile` для удобного управления проектом, а также добавить корневые конфигурации для TypeScript, линтеров и системы тестирования (Jest), чтобы подготовить базу для создания микросервисов.

## 🛑 Ограничения (CRITICAL)

1. **Makefile:** Отступы в `Makefile` должны быть сделаны СТРОГО табуляцией (Tab), а не пробелами. Иначе файл сломается.
2. **Атомарность:** Не создавай никакой бизнес-логики или папок микросервисов. Работай только с конфигурационными файлами в корне проекта.

---

## 📋 Шаги выполнения (Step-by-Step)

- [ ] **Шаг 1: Создание файлов игнорирования**
  - Создай корневой `.gitignore`. Обязательно включи туда: `node_modules`, `dist`, `build`, `.env`, `coverage`, `.turbo`, `.DS_Store`, `.idea`, `.vscode`.
  - Создай корневой `.dockerignore`. Он должен исключать всё, что не нужно для сборки образов: `.git`, `node_modules`, `docs`, `coverage`, `.turbo`, локальные `.env` файлы, `README.md`.

- [ ] **Шаг 2: Создание Makefile**
  - Создай файл `Makefile` в корне проекта.
  - Добавь команды для Docker: `docker-up` (поднимает `docker-compose.yml` в фоне), `docker-down` (останавливает и удаляет контейнеры), `docker-logs` (показывает логи всех сервисов).
  - Добавь команды для Turborepo: `build-all`, `lint-all`, `dev-all`, `test-all`, `typecheck-all`.
  - Добавь команду `help`, которая выводит список всех доступных команд с кратким описанием.

- [ ] **Шаг 3: Корневые конфигурации TypeScript и Линтеров**
  - Создай `tsconfig.base.json` со строгими правилами (`"strict": true`, `"noImplicitAny": true`, `"esModuleInterop": true`, `"skipLibCheck": true`).
  - Создай базовый `.eslintrc.js` (или `.json`) с правилами для TypeScript и Prettier.
  - Создай `.prettierrc` для единого стиля форматирования кода.

- [ ] **Шаг 4: Настройка Jest**
  - Создай корневой `jest.config.js` (или `jest.preset.js`), который будет служить базой для всех микросервисов.
  - Убедись, что конфиг поддерживает TypeScript (например, через `ts-jest` или `@swc/jest`).
  - Добавь необходимые зависимости (`jest`, `ts-jest`, `@types/jest`, `typescript`) в `package.json` корня проекта (в `devDependencies`). Выполни `yarn install`.

- [ ] **Шаг 5: Проверка turbo.json**
  - Проверь корневой `turbo.json`. Убедись, что в нем прописан пайплайн для команды `test` (например, `"test": { "dependsOn": ["^build"], "outputs": ["coverage/**"] }`).

- [ ] **Шаг 6: Проверка и Отчет (CRITICAL)**
  - Убедись, что команда `make help` работает и выводит список команд без ошибок форматирования.
  - Создай файл-отчет `docs/tasks/reports/001-fixup-configs.done.md` согласно правилам нашего пайплайна (`03-execution-pipeline.md`).
  - В отчете перечисли созданные файлы и предложи команду для коммита (например, `chore: add root configs, makefile and ignore files`).
