# Отчёт 016.3: Layout microservicesAccess

**Дата:** 2026-04-23

**Тип задачи:** PERMISSIONS_001 infrastructure

## Цель

Добавить проп `microservicesAccess` в UiKit Layout для правильного отображения заблокированных сервисов в навигации.

---

## Выполненные изменения

### 1. Layout.tsx — добавлен проп `microservicesAccess`

**Файл:** `libs/ui-kit/src/components/layout/Layout.tsx`

Добавлен новый проп `microservicesAccess?: Record<string, boolean>` и логика его обработки:

**Для заблокированного сервиса** (`microservicesAccess[serviceId] === false`):
- ОДИН пункт меню (parent), БЕЗ детей
- Иконка: `bi bi-lock`
- `locked: true`
- `path: \`/${m.serviceId}\`` — ведёт на `/pulse`, `/service` и т.д.

**Для доступного сервиса** (`microservicesAccess[serviceId] !== false`):
- Parent с children (как раньше)
- Иконка: `bi bi-grid`
- Дети — как обычно

### 2. Обновлены тесты

**Файл:** `libs/ui-kit/src/components/layout/Layout.spec.tsx`

Добавлены тесты для `microservicesAccess`:

| Тест | Описание |
|------|----------|
| marks service as locked when false | Проверяет `locked: true` |
| sets path to /{serviceId} for locked | Проверяет path `/pulse` |
| sets bi bi-lock icon for locked | Проверяет иконку |
| has no children for locked service | Заблокированный без детей |
| keeps children for accessible service | Доступный с детьми |
| uses bi bi-grid for accessible | Иконка для доступного |
| shows all accessible without prop | Без пропа все доступны |
| marks multiple locked | Несколько заблокированных |
| treats empty object as all accessible | Пустой объект = все доступны |
