# Task infrastructure/006: Обновление TypeScript и реорганизация задач

## 🎯 Цель

Обновить TypeScript, устранить предупреждение об устаревшем `moduleResolution=node10` с помощью флага `ignoreDeprecations`, а также навести порядок в документации: убрать числовые префиксы у папок микросервисов и рассортировать все текущие и будущие задачи по датам их создания.

## 📋 Задачи для выполнения

### 1. Обновление TypeScript и фикс `tsconfig.json`

- Обновить пакет `typescript` до последней стабильной версии во всем монорепозитории (в корневом `package.json` и везде, где он указан).
- Найти все файлы `tsconfig.json` и `tsconfig.base.json` в проекте.
- **НЕ МЕНЯТЬ** параметр `moduleResolution`.
- Добавить в секцию `compilerOptions` параметр `"ignoreDeprecations": "6.0"`, чтобы отключить предупреждение о `moduleResolution=node10`.
- Убедиться, что после обновления TS и добавления флага команда `npx tsc --noEmit` (или `yarn typecheck`) проходит без ошибок и предупреждений во всех проектах.

### 2. Реорганизация структуры папок и задач

- **Переименование папок доменов:** Убрать числовые префиксы у папок микросервисов/доменов в директории `docs/tasks/`.
  - `000-infrastructure` -> `infrastructure`
  - `100-hub` -> `hub`
  - И так далее для всех существующих папок.
- **Сортировка по датам:** Определить дату создания каждой задачи (можно взять из названия отчета, из текста самой задачи или по дате файла).
- Создать внутри новых папок доменов подпапки с датами в формате `YYYY-MM-DD`.
- Переместить файлы задач и соответствующие им отчеты в эти папки.
  _Пример было:_ `docs/tasks/100-hub/004-2-user-entity-contracts.md`
  _Пример стало:_ `docs/tasks/hub/2026-04-16/004-2-user-entity-contracts.md`
- **КРИТИЧЕСКИ ВАЖНО:** Нумерация самих файлов (001, 002, 003...) внутри имени файла должна остаться неизменной! Сквозная нумерация продолжается в рамках микросервиса, независимо от папки с датой.

### 3. Фиксация глобального правила

- Создать или обновить файл `docs/tasks/README.md` (или `docs/RULES.md`, если он есть).
- Дописать туда новое глобальное правило:
  > **Правило хранения задач:** Все новые задачи (`.md`) и отчеты о выполнении (`.report.md` / `.done.md`) должны сохраняться в директорию, соответствующую дате создания задачи. Имена папок доменов не должны содержать числовых префиксов. Формат пути: `docs/tasks/<domain>/<YYYY-MM-DD>/<number>-<task-name>.md`. Нумерация задач остается сквозной для каждого домена.

## 🏁 Критерии приемки (DoD)

1. Версия `typescript` обновлена до последней.
2. В редакторе и при запуске `tsc` больше нет предупреждения о `moduleResolution=node10` благодаря `"ignoreDeprecations": "6.0"`.
3. Сборка и проверка типов не сломались.
4. Папки доменов переименованы (без префиксов `000-`, `100-`).
5. Все файлы в `docs/tasks/` лежат в папках с датами.
6. Нумерация существующих файлов задач не изменилась.
7. Добавлено глобальное правило в документацию.
8. Написан отчет о выполнении задачи (уже по новому пути, в папке с сегодняшней датой `2026-04-17`).
