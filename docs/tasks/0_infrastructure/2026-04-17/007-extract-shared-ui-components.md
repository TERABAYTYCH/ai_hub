# Задача 007: Вынесение общих UI-компонентов в libs/ui-kit

## Контекст
В процессе создания микросервиса Pulse мы реализовали для него собственные компоненты Layout (Header, Sidebar) и страницы авторизации/регистрации. В результате появилось дублирование кода между `apps/hub/frontend` и `apps/pulse/frontend`. 
Для обеспечения визуальной консистентности платформы и упрощения поддержки необходимо вынести эти базовые элементы в общую библиотеку `libs/ui-kit`, сделав их максимально переиспользуемыми.

**ВНИМАНИЕ:** При рефакторинге React-компонентов обязательно использовать скилл `vercel-react-best-practices`.

## Требования и шаги реализации

### 1. Вынесение Layout-компонентов (Основа: Hub)
- Взять за основу компоненты `AppLayout`, `Header` и `Sidebar` из **`apps/hub/frontend`**.
- Перенести их в `libs/ui-kit/src/components/layout/`.
- Сделать их "глупыми" (dumb) и настраиваемыми:
  - `Sidebar` должен принимать массив элементов навигации (ссылка, иконка, текст) через пропсы, чтобы каждый микросервис мог передать свое меню.
  - `Header` должен принимать пропсы для отображения данных пользователя, коллбек для Logout и (опционально) слот для кастомных элементов.
  - `AppLayout` должен выступать просто оберткой, принимающей `Sidebar`, `Header` и `children` (контент).

### 2. Вынесение компонентов авторизации (Основа: Pulse)
- Взять за основу UI-часть форм из `LoginPage.tsx` и `RegisterPage.tsx` микросервиса **`apps/pulse/frontend`**.
- Вынести их в `libs/ui-kit/src/components/auth/` (например, `LoginForm` и `RegisterForm`).
- Формы не должны содержать логики API-запросов! Они должны принимать пропсы: `onSubmit` (с введенными данными), `isLoading` (для блокировки кнопок и показа спиннера) и `error` (для отображения ошибки).

### 3. Настройка экспорта в ui-kit
- Убедиться, что все новые компоненты и их TypeScript-интерфейсы экспортируются в `libs/ui-kit/src/index.ts`.

### 4. Рефакторинг Hub Frontend
- Удалить локальные файлы `Header`, `Sidebar`, `AppLayout` в `apps/hub/frontend`.
- Заменить их на импорты из общей библиотеки (например, `import { AppLayout, Sidebar } from '@kremen/ui-kit'`).
- Настроить передачу нужных пропсов (меню Хаба, данные пользователя).
- Обновить страницы логина и регистрации Хаба на использование общих компонентов `LoginForm` и `RegisterForm`.

### 5. Рефакторинг Pulse Frontend
- Удалить локальные файлы `Header`, `Sidebar`, `AppLayout` в `apps/pulse/frontend`.
- Заменить их на импорты из `ui-kit`.
- Передать в `Sidebar` специфичное меню Pulse (Dashboard, Metrics, Alerts, Settings).
- Обновить страницы логина и регистрации Pulse на использование общих компонентов `LoginForm` и `RegisterForm`.

## Критерии приемки (DoD)
1. Компоненты `AppLayout`, `Header`, `Sidebar`, `LoginForm`, `RegisterForm` находятся исключительно в `libs/ui-kit`.
2. В папках `apps/hub/frontend` и `apps/pulse/frontend` нет дублирующегося кода этих компонентов.
3. При локальном запуске Hub Frontend интерфейс и авторизация работают без изменений.
4. При локальном запуске Pulse Frontend интерфейс и авторизация работают без изменений, отображается специфичное меню Pulse.
5. Module Federation (встраивание Pulse в Hub) продолжает работать корректно (экспортируется только контентная часть, без двойного Layout).
6. Сборка всех проектов (`turbo build`) проходит без ошибок типизации.
