# Task 000-infrastructure/004: Внедрение политики временных меток для отчетов

## 🎯 Цель

Изменить стандарт именования файлов отчетов (`*.report.md`), добавив в них обязательную временную метку. Переименовать все существующие файлы отчетов в проекте под новый стандарт и обновить глобальные инструкции, чтобы в будущем это правило применялось автоматически.

## 🧭 Контекст

Для лучшего отслеживания истории выполнения задач и предотвращения перезаписи отчетов с одинаковыми именами (если задача выполняется в несколько подходов), мы вводим новый стандарт именования. Теперь каждый отчет должен заканчиваться временной меткой: `YYYYMMDD-HHMM.md`.

## 📋 Задачи для выполнения

### 1. Обновление документации (Пайплайн)

- Открой файл `03-execution-pipeline.md` (и `AGENT.md`, если применимо).
- Найди раздел, описывающий создание отчета по завершении задачи.
- Добавь строгое правило: "Все файлы отчетов должны иметь формат `[название-задачи].report.[YYYYMMDD-HHMM].md`. Например: `001-init.report.20260416-1930.md`. Использование формата без даты запрещено."

### 2. Миграция существующих отчетов

- Найди все файлы с маской `*.report.md` в директории `docs/tasks/` и её подпапках.
- Прочитай содержимое каждого отчета, чтобы найти дату его выполнения (обычно указана внутри в формате "15 апреля 2026").
- Переименуй эти файлы, добавив временную метку.
  - Если точное время неизвестно, используй время создания файла из файловой системы (через `fs.stat` в Node.js или `stat` в bash) или просто добавь `YYYYMMDD-0000`.
  - Пример: `001-init.report.md` -> `001-init.report.20260415-1200.md`.
- Убедись, что ссылки на эти отчеты в других markdown-файлах (если они есть) также обновлены.

## 🛠 Требования и ограничения

- Для массового переименования рекомендуется написать небольшой Node.js или Bash скрипт, выполнить его, а затем удалить.
- Убедиться, что ни один старый отчет не потерян.
- **Сам отчет об этой задаче** должен быть сохранен уже по новому правилу (например, `004-reports-timestamp-policy.report.20260416-1935.md`).

## 🏁 Критерии приемки (DoD)

1. Файл `03-execution-pipeline.md` обновлен и содержит новое правило именования.
2. В папке `docs/tasks/` и её подпапках больше нет файлов, заканчивающихся просто на `.report.md` (без даты).
3. Все старые отчеты успешно переименованы с сохранением их содержимого.
4. Создан отчет о выполнении текущей задачи с использованием нового формата именования.
