# Task 100-hub/004: Управление мастер-базой устройств (Device CRUD)

## 🎯 Цель

Создать сущность `Device` в сервисе Hub, реализовать CRUD-операции на бэкенде (с защитой через JWT) и сделать страницу управления устройствами на фронтенде с использованием React Bootstrap.

## 🧭 Контекст

Сервис Hub является центральным узлом (Source of Truth) для всей платформы. Микросервисы Pulse (мониторинг), Service (обслуживание) и Control (управление) будут работать с одними и теми же устройствами. Поэтому именно в Hub мы создаем мастер-базу устройств. В будущем при создании нового устройства Hub будет отправлять событие в Kafka, чтобы другие сервисы узнали о нем, но пока нам нужен базовый REST API и UI.

## 📋 Задачи для выполнения

### 1. Общие контракты (`libs/contracts/src/hub/devices`)
- Создать директорию `devices` в контрактах Hub.
- Описать DTO:
  - `CreateDeviceDto` (поля: `name` (string), `type` (string/enum), `description` (optional string)).
  - `UpdateDeviceDto` (частичное обновление).
  - `DeviceResponseDto` (поля: `id`, `name`, `type`, `status`, `description`, `createdAt`).
- Экспортировать их через `index.ts`.

### 2. Backend (NestJS - `apps/hub/backend`)
- Создать сущность TypeORM `Device`.
  - Поля: `id` (UUID, primary), `name` (varchar), `type` (varchar), `status` (enum: 'ACTIVE', 'INACTIVE', 'MAINTENANCE' - по умолчанию 'ACTIVE'), `description` (text, nullable), `createdAt`, `updatedAt`.
- Сгенерировать и применить миграцию для создания таблицы `devices` (не использовать `synchronize: true`).
- Создать `DeviceModule`, `DeviceService`, `DeviceController`.
- Реализовать эндпоинты (все должны быть защищены `JwtAuthGuard`):
  - `GET /api/devices` — получение списка всех устройств.
  - `GET /api/devices/:id` — получение устройства по ID.
  - `POST /api/devices` — создание нового устройства.
  - `PATCH /api/devices/:id` — обновление устройства (например, смена статуса).
  - `DELETE /api/devices/:id` — удаление устройства.

### 3. Frontend (React - `apps/hub/frontend`)
- Создать сервис `devices.api.ts` для взаимодействия с API (не забыть передавать JWT токен в заголовке `Authorization: Bearer ...`).
- Создать страницу `DevicesPage.tsx`.
- Использовать компоненты `react-bootstrap` (Table, Card, Badge для статусов, Button).
- Реализовать отображение списка устройств в виде таблицы или сетки карточек.
- Добавить кнопку "Добавить устройство", которая открывает Bootstrap Modal с формой создания (имя, тип, описание).
- Добавить эту страницу в роутинг (например, `/devices`) и сделать ссылку на нее в навигационной панели (Navbar), которая должна отображаться только для авторизованных пользователей.

## 🛠 Требования и ограничения
- **Strict Mode:** TypeScript в строгом режиме, без `any`.
- **Авторизация:** Ни один эндпоинт устройств не должен быть доступен без валидного JWT токена.
- **UI/UX:** Использовать общую тему из `libs/ui-kit` (которую мы сделали в задаче 000-infrastructure/003). Таблица/карточки должны корректно смотреться как в светлой, так и в темной теме.

## 🏁 Критерии приемки (DoD)
1. Миграция для таблицы `devices` успешно создана и применена.
2. CRUD API для устройств работает и требует авторизации (возвращает 401 без токена).
3. На фронтенде авторизованный пользователь может зайти на `/devices`.
4. Пользователь может создать новое устройство через модальное окно.
5. Созданное устройство сразу отображается в списке.
6. Написан и сохранен отчет с использованием нового правила таймстемпов (формат: `004-device-management.report.[YYYYMMDD-HHMM].md`).
