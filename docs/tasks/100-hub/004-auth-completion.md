# Task 100-hub/004: Завершение флоу авторизации (Admin Seed & Register Page)

## 🎯 Цель

Обеспечить возможность входа в систему: реализовать автоматическое создание учетной записи администратора при старте бэкенда и добавить страницу регистрации на фронтенде.

## 🧭 Контекст

В предыдущей задаче была реализована JWT-авторизация и страница логина. Однако сейчас разработчик не может войти в систему, так как в базе нет пользователей, а UI для регистрации отсутствует. Нам нужен механизм автоматического создания (seeding) админского аккаунта через переменные окружения, а также полноценная страница регистрации для новых пользователей.

## 📋 Задачи для выполнения

### 1. Backend (NestJS - `apps/hub/backend`)
- Добавить в `.env.example` и `.env` (через скрипт или инструкцию) переменные:
  - `ADMIN_EMAIL=admin@ject.hub`
  - `ADMIN_PASSWORD=admin`
- Реализовать логику Seeding'а:
  - Использовать хук жизненного цикла `OnApplicationBootstrap` (например, в `AuthModule` или `UsersModule`, если он есть).
  - При старте приложения проверять, существует ли пользователь с email из `ADMIN_EMAIL`.
  - Если нет — создавать его, обязательно хешируя пароль из `ADMIN_PASSWORD` через `bcrypt`.

### 2. Frontend (React - `apps/hub/frontend`)
- Создать страницу `RegisterPage.tsx` с использованием React Bootstrap.
- Форма должна содержать поля: Email, Password, Confirm Password.
- Добавить валидацию на совпадение паролей перед отправкой.
- Интегрировать вызов API регистрации (эндпоинт `POST /api/auth/register`, созданный в задаче 003).
- При успешной регистрации перенаправлять пользователя на страницу `/login` (или сразу авторизовывать и кидать на главную).
- Добавить ссылки-переходы между `LoginPage` ("Нет аккаунта? Зарегистрироваться") и `RegisterPage` ("Уже есть аккаунт? Войти").

## 🛠 Требования и ограничения
- **Strict Mode:** TypeScript в строгом режиме.
- **Безопасность:** Пароль админа ОБЯЗАТЕЛЬНО должен хешироваться перед сохранением в БД, иначе логин не сработает (т.к. `validateUser` сравнивает хеши).
- **UI/UX:** Использовать компоненты Bootstrap (Container, Card, Form, Button), чтобы страница регистрации стилистически совпадала со страницей логина и поддерживала темную тему.

## 🏁 Критерии приемки (DoD)
1. При запуске бэкенда в пустой БД автоматически создается пользователь-админ.
2. Можно успешно авторизоваться под админом на странице `/login`.
3. На фронтенде доступна страница `/register`.
4. Регистрация нового пользователя через UI работает корректно.
5. Написан отчет в формате `004-auth-completion.report.[YYYYMMDD-HHMM].md`.
