# SERVICE_001.1: Backend — Структура и копирование из Pulse

## Цель

Создать структуру backend для микросервиса **Service** (Обслуживание) путём копирования из Pulse и адаптации.

## Задачи

### 1. Копирование структуры

Скопировать директорию `apps/pulse/backend/` в `apps/service/backend/`.

### 2. Изменения в скопированных файлах

| Файл | Изменение |
|------|-----------|
| `package.json` | name: `@ject-hub/service-backend` |
| `src/main.ts` | порт: `3002`, БД: `mysql-service` |
| `src/app.module.ts` | ОСТАВИТЬ HubProxyModule (получение устройств из Hub) |

### 3. Проверки (Asserts)

- [ ] Директория `apps/service/backend/` создана
- [ ] `package.json` содержит `@ject-hub/service-backend`
- [ ] `src/main.ts` слушает порт 3002
- [ ] HubProxyModule импортирован в app.module

### 4. Проверки на отсутствие хвостов Pulse

- [ ] `grep -r "pulse" apps/service/backend/` — 0 результатов
- [ ] `grep -r "pulse" apps/service/backend/package.json` — 0 результатов
- [ ] `grep -r "pulse" apps/service/backend/src/` — 0 результатов
- [ ] `grep -r "3001" apps/service/backend/` — 0 результатов (захардкоженные порты pulse)
- [ ] `grep -r "mysql-pulse" apps/service/backend/` — 0 результатов

## План

1. Скопировать `apps/pulse/backend/` → `apps/service/backend/`
2. Изменить `package.json` — name, workspace reference
3. Изменить `src/main.ts` — порт 3002, DATABASE_NAME=mysql-service
4. Проверить HubProxyModule в app.module.ts (должен остаться)
5. Запустить проверки (asserts и grep на хвосты)
6. Запушить в ветку `service_init`
