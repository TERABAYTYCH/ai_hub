# 001.2: Frontend — Структура и копирование из Pulse

## Цель

Создать frontend для микросервиса Service путём копирования из Pulse и адаптации.

## Задачи

### 1. Копирование структуры

Скопировать `apps/pulse/frontend/` → `apps/service/frontend/`

### 2. Изменения в скопированных файлах

| Файл | Изменение |
|------|-----------|
| `package.json` | name: `@ject-hub/service-frontend`, порт: `5175` |
| `vite.config.ts` | name: `'service'`, exposes: Dashboard, Devices, Settings |
| `.env` | VITE_API_URL=http://api.service.lvh.me |
| manifestPlugin | иконки: bi bi-wrench (Dashboard), bi bi-grid (Devices), bi bi-gear (Settings) |
| serveDistAssetsPlugin | оставить как у Pulse |

### 3. Проверки (Asserts)

- [ ] Директория `apps/service/frontend/` создана
- [ ] `package.json` содержит `@ject-hub/service-frontend`
- [ ] `vite.config.ts` exposes: Dashboard, Devices, Settings
- [ ] `serveDistAssetsPlugin` скопирован и работает
- [ ] manifest.json содержит serviceId: "service"
- [ ] manifest.json содержит navigation: Dashboard, Devices, Settings

### 4. Проверки на отсутствие хвостов Pulse

- [ ] `grep -r "pulse" apps/service/frontend/` — 0 результатов
- [ ] `grep -r "3001\|5174" apps/service/frontend/` — 0 результатов
- [ ] `grep -r "pulse.lvh.me" apps/service/frontend/` — 0 результатов
- [ ] `grep -r "Pulse" apps/service/frontend/src/` — 0 результатов

## План

1. Скопировать `apps/pulse/frontend/` → `apps/service/frontend/`
2. Изменить `package.json` — name, port
3. Изменить `vite.config.ts` — name, exposes, manifestPlugin
4. Изменить `.env` — VITE_API_URL
5. Проверить HubProxyModule в app.module.ts (должен остаться)
6. Запустить проверки (asserts и grep на хвосты)
7. Запушить в ветку `service_init`
