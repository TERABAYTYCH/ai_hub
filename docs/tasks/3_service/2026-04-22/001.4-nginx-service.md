# 001.4: Nginx — Добавить Service server blocks

## Цель

Добавить server blocks для Service в nginx.conf.

## Задачи

### 1. Добавить в nginx.conf

**service.lvh.me (Frontend):**
```nginx
server {
    listen 80;
    server_name service.lvh.me;

    location / {
        proxy_pass http://service-frontend:5175;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
```

**api.service.lvh.me (Backend):**
```nginx
server {
    listen 80;
    server_name api.service.lvh.me;
    location / {
        proxy_pass http://service-backend:3002;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

### 2. Проверки

- [ ] `nginx -t` проходит
- [ ] service.lvh.me проксируется к Vite
- [ ] api.service.lvh.me проксируется к backend

### 3. Проверки на отсутствие хвостов Pulse

- [ ] Нет упоминаний pulse.lvh.me или api.pulse.lvh.me в service блоках
- [ ] service.lvh.me и api.service.lvh.me корректно настроены

## План

1. Прочитать текущий nginx.conf
2. Добавить service.lvh.me server block
3. Добавить api.service.lvh.me server block
4. Запустить nginx -t
5. Перезапустить nginx
