**Дата:** 2026-04-23  
**Домен:** 3_service  
**Родительский план:** PERMISSIONS_001 (Управление доступом к микросервисам)

---

## Цель

Аналогично 006.1 для Pulse: добавить `/lock` route и глобальную блокировку Service.

---

## Контекст

Аналогично Pulse Frontend. При прямом входе на `service.lvh.me/lock` — LockPage. При `microservices.service === false` — все routes редиректятся на `/lock`.

---

## Требуемое состояние

### 1. App.tsx — структура аналогичная Pulse

**Файл:** `apps/service/frontend/src/App.tsx`

- Добавить импорт `LockPage`, `useAuth`
- Разделить на компоненты: `LoadingScreen`, `AppRoutes`, `App` (Rules of Hooks)
- `useAuth()` вызывается только в `App()` первым
- `isServiceLocked` проверяется в `AppRoutes()`

### 2. AppRoutes — lock routes

При `isServiceLocked`:
- Все routes редиректятся на `/lock` через `/*` wildcard
- Конкретный route `/lock` → `<Layout><LockPage serviceName="Service" /></Layout>`

При НЕ `isServiceLocked`:
- Static routes: `/`, `/login`, `/register`, `/dashboard`, `/devices`, `/settings`
- Catch-all `/*` → `/dashboard`

### 3. useDynamicRoutesConfig — блокировка других сервисов

Для manifest с `isLocked`:
- Конкретный route `/${serviceId}/lock` → `<ProtectedRoute><Layout><LockPage serviceName={manifest.name} /></Layout></ProtectedRoute>`
- Wildcard `/${serviceId}/*` → Navigate на `/${serviceId}/lock`

### 4. Layout.tsx — меню при блокировке

**Файл:** `apps/service/frontend/src/components/layout/Layout.tsx`

- Использовать `useAuth()` для проверки `microservicesAccess['service']`
- Если заблокирован: только `lockedMenuItem` с путём `/lock`
- `excludeServices = isServiceLocked ? ['service', 'pulse'] : ['service']`
- `hubSettingsItem = isServiceLocked ? undefined : {...}`
