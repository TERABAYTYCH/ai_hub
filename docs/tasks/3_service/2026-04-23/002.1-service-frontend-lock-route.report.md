## Что сделано

### 1. App.tsx — структура с lock routes

**Файл:** `apps/service/frontend/src/App.tsx`

- Добавлены импорты `LockPage`, `useAuth`
- Разделены компоненты: `LoadingScreen`, `AppRoutes`, `App`
- `useAuth()` вызывается только в `App()` первым (Rules of Hooks)
- `isServiceLocked` проверяется в `AppRoutes()`

### 2. AppRoutes — lock routes

При `isServiceLocked`:
- `/*` → `/lock`
- `/lock` → `<Layout><LockPage serviceName="Service" /></Layout>`

При НЕ `isServiceLocked`:
- Static routes + catch-all `/*` → `/dashboard`

### 3. useDynamicRoutesConfig — блокировка других сервисов

Для `isLocked` сервисов:
- `/${serviceId}/lock` → `<ProtectedRoute><Layout><LockPage /></Layout></ProtectedRoute>`
- `/${serviceId}/*` → Navigate на `/${serviceId}/lock`

### 4. Layout.tsx — меню при блокировке

**Файл:** `apps/service/frontend/src/components/layout/Layout.tsx`

- Добавлен `useAuth()` для проверки `microservicesAccess['service']`
- Если заблокирован: `lockedMenuItem` с путём `/lock`
- `excludeServices = isServiceLocked ? ['service', 'pulse'] : ['service']`
- `hubSettingsItem = isServiceLocked ? undefined : {...}`
