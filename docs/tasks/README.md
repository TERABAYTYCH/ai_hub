# Документация задач

## Глобальные правила

### Правило хранения задач

Все новые задачи (`.md`) и отчеты о выполнении (`.report.[HHMM].md`) должны сохраняться в директорию, соответствующую дате создания задачи.

- Имена папок доменов используют формат: `<индекс>_<домен>` (например, `0_infrastructure`, `1_hub`)
- Формат пути: `docs/tasks/<индекс>_<домен>/<YYYY-MM-DD>/<номер>-<название-задачи>.report.[HHMM].md`
- Нумерация задач остается сквозной для каждого домена

Порядок доменов:

- `0_infrastructure` - Задачи инфраструктуры
- `1_hub` - Задачи микросервиса Hub
- `2_pulse` - Задачи микросервиса Pulse
- `3_service` - Задачи микросервиса Service
- `4_control` - Задачи микросервиса Control

Пример:

- До: `docs/tasks/100-hub/004-2-user-entity-contracts.md`
- После: `docs/tasks/1_hub/2026-04-16/004-2-user-entity-contracts.report.2115.md`
