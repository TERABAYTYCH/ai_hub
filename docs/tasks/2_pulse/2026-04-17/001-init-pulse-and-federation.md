# Задача 007: Инициализация микросервиса Pulse и настройка Module Federation

## Контекст

Мы приступаем к реализации главной архитектурной фичи нашей платформы — микрофронтендов. Необходимо создать второй микросервис **Pulse** (Мониторинг) и настроить бесшовное встраивание его фронтенда в наше основное приложение **Hub** с использованием Vite Module Federation.
_Примечание: Интеграцию с Kafka пока откладываем, фокус исключительно на связке фронтендов и базовом роутинге._

## Требования и шаги реализации

### 1. Инициализация бэкенда Pulse (NestJS)

- Создать базовое приложение NestJS в папке `apps/pulse/backend`.
- Настроить порт приложения (например, `3001` для внутреннего использования в Docker).
- Создать простой контроллер (например, `AppController` с эндпоинтом `GET /api/pulse/health`, возвращающим статус `OK`).
- Добавить `pulse-backend` в корневой `docker-compose.yml`.
- Обновить конфигурацию Nginx (`nginx/nginx.conf` или аналогичный файл), чтобы запросы на `http://localhost/pulse/api/*` проксировались на контейнер `pulse-backend`.

### 2. Инициализация фронтенда Pulse (Vite + React)

- Создать базовое приложение React + Vite в папке `apps/pulse/frontend`.
- Установить плагин для Module Federation (например, `@originjs/vite-plugin-federation`).
- Настроить `vite.config.ts`:
  - Имя модуля: `pulse`.
  - Экспортировать базовый компонент страницы (например, `PulseDashboard.tsx`), который будет содержать простой заголовок "Pulse Dashboard" и карточку из нашего `ui-kit`.
  - Настроить сборку так, чтобы генерировался `remoteEntry.js`.
- Добавить `pulse-frontend` в `docker-compose.yml`.
- Настроить Nginx для отдачи статики фронтенда Pulse (например, по пути `/pulse` или на отдельном порту, чтобы Hub мог скачивать `remoteEntry.js`).

### 3. Настройка Hub Frontend (Host)

- Добавить плагин Module Federation в `apps/hub/frontend/vite.config.ts`.
- Настроить `remotes`: указать ссылку на `remoteEntry.js` микросервиса Pulse (через Nginx или напрямую через порт контейнера, в зависимости от локальной настройки).
- В роутере Hub (`react-router-dom`) добавить новый маршрут `/pulse`.
- Настроить ленивую загрузку (lazy load) компонента `PulseDashboard` из удаленного модуля (Remote).
- Обернуть удаленный компонент в `React.Suspense` и `ErrorBoundary`.
- Добавить ссылку "Мониторинг (Pulse)" в боковое меню (`Sidebar`) внутри `AppLayout`.

### 4. Инфраструктура и связи

- Убедиться, что `apps/pulse/frontend` и `apps/pulse/backend` добавлены в `package.json` воркспейса (Yarn Workspaces / Turborepo).
- Настроить скрипты `dev` и `build` в `turbo.json` для новых проектов.

## Критерии приемки (DoD)

1. При запуске `docker-compose up` поднимаются контейнеры Hub (фронт/бэк) и Pulse (фронт/бэк).
2. Nginx корректно маршрутизирует запросы:
   - `/pulse/api/health` отвечает 200 OK от бэкенда Pulse.
3. При входе в приложение Hub в боковом меню есть пункт "Pulse".
4. При клике на "Pulse" URL меняется на `/pulse`, и в контентной области `AppLayout` (без перезагрузки страницы) отображается компонент `PulseDashboard`, подгруженный из микросервиса Pulse.
5. В консоли браузера нет ошибок CORS или проблем с загрузкой `remoteEntry.js`.
