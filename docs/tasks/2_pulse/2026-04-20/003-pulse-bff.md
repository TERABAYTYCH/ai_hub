# Task 003: Создание BFF для Pulse и интеграция с Hub

Нам необходимо реализовать бэкенд для приложения Pulse, который будет выступать в роли прокси (BFF) для получения списка устройств из основного сервиса Hub. Запросы должны ходить по внутренней сети Docker с пробросом токена авторизации.

---

## 📋 Метаданные задачи

| **Атрибут** | **Значение** |
|-------------|--------------|
| **ID задачи** | 011 |
| **Целевой сервис** | `pulse-backend` |
| **Тип задачи** | Feature / BFF (Backend-For-Frontend) |

---

## 🛠 Инструкция для Разработчика (`code-terych-sub`)

1. **Настройка окружения:**
   - Убедитесь, что в `.env` файле бэкенда Pulse (и в `docker-compose.yml`) задана переменная для внутреннего обращения к Hub: `HUB_INTERNAL_API_URL=http://hub-backend:3000`.

2. **Создание эндпоинта `/devices`:**
   - Реализуйте контроллер в `pulse-backend`, который слушает `GET /devices` (без префикса `/api`).
   - При получении запроса бэкенд должен извлечь заголовок `Authorization` (токен) из входящего HTTP-запроса.
   - Сделайте HTTP-запрос (через Axios/Fetch/HttpModule) к `${HUB_INTERNAL_API_URL}/devices`, передав извлеченный заголовок `Authorization`.
   - Верните полученные данные обратно фронтенду Pulse.

3. **Фиксация изменений (Git):**
   - Сделайте коммит строго по нашему стандарту именования.
   - **Команда:** `git commit -m "Task 003: Создание BFF для Pulse - добавлен эндпоинт /devices с пробросом авторизации в Hub"`

---

## 🕵️‍♂️ Инструкция для Тестировщика (`test-engineer`)

1. **Сборка и запуск:**
   - Пересоберите контейнер бэкенда Pulse: `docker-compose up -d --build pulse-backend`.

2. **Проверка без авторизации:**
   - Сделайте запрос `GET http://api.pulse.localhost/devices` **без токена**. 
   - *Ожидаемый результат:* `401 Unauthorized`.

3. **Проверка успешного запроса:**
   - Авторизуйтесь через фронтенд или получите валидный токен.
   - Сделайте запрос `GET http://api.pulse.localhost/devices` **с токеном** в заголовке `Authorization`.
   - *Ожидаемый результат:* `200 OK` и JSON со списком устройств.

4. **Проверка UI:**
   - Откройте фронтенд Pulse и убедитесь, что список устройств успешно рендерится на странице.
