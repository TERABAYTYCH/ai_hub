# Task 04: Вынос панели устройств в UI Kit и интеграция во фронтенд Pulse

На данный момент во фронтенде Pulse отсутствует функционал отображения списка устройств, в то время как в Hub уже есть готовая панель. Наша цель — избежать дублирования кода, вынести визуальные компоненты панели в общую библиотеку (UI Kit) и собрать на их основе интерфейс для Pulse, подключив его к нашему новому BFF.

## 🛠 Инструкция для Разработчика (`code-terych-sub`)

**⚠️ КРИТИЧЕСКОЕ НАПОМИНАНИЕ:** Активируй свой внутренний скилл `ux ui pro`. При переносе и создании компонентов уделяй максимальное внимание отступам, типографике, состояниям загрузки (скелетоны/спиннеры), пустым состояниям (empty states) и отзывчивости интерфейса. Панель должна выглядеть премиально и работать плавно.

1. **Экстракция в UI Kit:**
   - Найди компоненты панели списка устройств во фронтенде Hub.
   - Выдели "глупые" (presentational) компоненты (карточка устройства, список, индикаторы статуса) и перенеси их в общую библиотеку `ui-kit`.
   - Убедись, что компоненты в `ui-kit` не привязаны к бизнес-логике конкретного приложения (принимают данные через `props`, отдают события через `callbacks`).

2. **Рефакторинг Hub:**
   - Обнови фронтенд Hub: замени локальные компоненты панели устройств на импорты из `ui-kit`.
   - Убедись, что существующий функционал Hub не сломался.

3. **Интеграция в Pulse:**
   - Создай новую страницу/модуль во фронтенде Pulse для отображения устройств.
   - Используй перенесенные компоненты из `ui-kit` для построения UI.
   - Настрой получение данных: фронтенд Pulse должен делать запрос к локальному BFF (`/api/devices`), который мы реализовали в предыдущей задаче.

4. **Фиксация изменений (Git):**
   - **Команда:** `git commit -m "Task 04: Вынос панели устройств в UI Kit и реализация списка устройств в Pulse через BFF"`

## 🕵️‍♂️ Инструкция для Тестировщика (`test-engineer`)

1. **Регресс Hub:**
   - Открой фронтенд Hub и убедись, что список устройств отображается корректно, стили не поехали, а интерактивные элементы работают как раньше.

2. **Проверка Pulse (Новая фича):**
   - Открой фронтенд Pulse.
   - Перейди в новый раздел устройств.
   - *Ожидаемый результат:* Панель устройств выглядит идентично панели в Hub.
   - Открой Network Tab: убедись, что данные запрашиваются с бэкенда Pulse (`/api/devices`), а не напрямую из Hub.

3. **Проверка UX/UI:**
   - Проверь поведение при медленном интернете (должны быть красивые лоадеры).
   - Проверь отображение при пустом списке устройств (понятный empty state).
