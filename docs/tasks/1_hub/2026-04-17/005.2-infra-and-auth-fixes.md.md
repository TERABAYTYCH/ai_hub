# Task 1_hub/005.2: Глобальный .gitignore и фикс роутинга авторизации

## 🎯 Цель
1. Настроить исчерпывающий `.gitignore` для монорепозитория, чтобы исключить попадание логов и кэша во всех вложенных папках.
2. Исправить логику фронтенда: неавторизованный пользователь не должен попадать на страницу `/devices`. При получении ошибки `401 Unauthorized` от любого API, приложение должно очищать токены и редиректить на страницу логина.

## 📋 Задачи для выполнения

### Часть 1: Инфраструктура (.gitignore)
1. Открыть корневой файл `.gitignore`.
2. Добавить глобальное игнорирование для папок `.turbo` на всех уровнях вложенности (использовать `**/.turbo` или просто `.turbo`).
3. Расширить `.gitignore` стандартными правилами: `node_modules`, папки сборок (`dist`, `build`, `out`), логи (`*.log`, `npm-debug.log*`), локальные `.env` файлы, артефакты IDE (`.DS_Store`, `.vscode/`, `.idea/`), кэш (`.eslintcache`, `*.tsbuildinfo`).
4. Выполнить `git rm -r --cached .` и затем `git add .`, чтобы Git перечитал правила и удалил из индекса вложенные `turbo-lint.log` и `turbo-typecheck.log`.

### Часть 2: Фронтенд (Auth Guard & Interceptor)
1. **Защита роутов (Protected Route):**
   - Убедиться, что роут `/devices` обернут в компонент защиты (например, `ProtectedRoute` или `RequireAuth`).
   - Если токена нет в `localStorage` (или стейте) при попытке зайти на `/devices`, сразу редиректить на `/login`.
2. **Глобальный перехватчик 401 (Interceptor):**
   - Если используется `axios`, добавить `axios.interceptors.response.use`. Если `fetch` — обновить базовую функцию запроса.
   - При получении статуса `401 Unauthorized` от бэкенда:
     - Удалить токен из `localStorage` (и очистить стейт пользователя, если есть).
     - Перенаправить пользователя на страницу `/login` (используя `useNavigate` или `window.location.href`).
3. Проверить сценарий: зайти на `/devices` с валидным токеном, затем вручную удалить токен из Application -> Local Storage в DevTools и обновить страницу (или сделать действие). Приложение должно выкинуть на логин.

## 🏁 Критерии приемки (DoD)
1. В `git status` чисто от логов и кэшей (включая вложенные `.turbo`).
2. Неавторизованный пользователь не может открыть страницу `/devices`.
3. При протухании токена (ошибка 401) происходит автоматический логаут и редирект на `/login`.
4. Написан краткий отчет `005.2-infra-and-auth-fixes.report.[HHMM].md`.
