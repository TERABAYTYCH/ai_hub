# Task 1_hub/005.3: Исправление редиректа после успешной авторизации

## 🎯 Цель
Настроить автоматический переход пользователя со страницы `/login` на главную страницу (или `/devices`) после успешного входа в систему.

## 📋 Задачи для выполнения
1. Открыть компонент страницы авторизации на фронтенде (вероятно, `apps/hub/frontend/src/pages/LoginPage.tsx` или аналогичный).
2. Найти обработчик успешного сабмита формы логина (там, где токен сохраняется в `localStorage` или стейт).
3. Добавить логику редиректа:
   - Использовать хук `useNavigate` из `react-router-dom`.
   - После успешного сохранения токена вызвать `navigate('/')` или `navigate('/devices')`.
4. (Опционально, но желательно) Если пользователь уже авторизован (токен есть в хранилище) и пытается зайти на страницу `/login`, его также должно сразу редиректить внутрь приложения, чтобы он не видел форму входа повторно.

## 🏁 Критерии приемки (DoD)
1. При вводе верных учетных данных на странице `/login` происходит успешный вход и мгновенный переход на защищенную страницу (например, `/devices`).
2. Авторизованный пользователь при попытке открыть `/login` автоматически перенаправляется внутрь приложения.
3. Написан краткий отчет `005.3-fix-login-redirect.report.[HHMM].md`.
