# Task 1_hub/005.1: Исправление авторизации в API устройств на фронтенде

## 🎯 Цель
Устранить ошибку `401 (Unauthorized)` при обращении к эндпоинтам `/api/devices` с фронтенда, добавив передачу JWT-токена в заголовки запросов.

## 📋 Задачи для выполнения
1. Открыть файл `apps/hub/frontend/src/api/devices.api.ts`.
2. Найти все функции, делающие HTTP-запросы (`getDevices`, `getDevice`, `createDevice`, `updateDevice`, `deleteDevice`).
3. Добавить логику получения токена (например, из `localStorage`, куда он сохраняется при логине).
4. Добавить заголовок `Authorization: Bearer <token>` в каждый запрос (в `fetch` или `axios`, смотря что используется).
5. Убедиться, что если токена нет, запрос либо не отправляется, либо корректно обрабатывается ошибка (редирект на страницу логина).
6. Проверить работоспособность страницы `Device Management` — список устройств должен загружаться без ошибок 401.

## 🏁 Критерии приемки (DoD)
1. В `devices.api.ts` реализована передача JWT-токена.
2. Страница `/devices` успешно загружает данные (или пустой список, если устройств нет) без ошибок 401 в консоли.
3. Создание, обновление и удаление устройств также работают корректно и проходят авторизацию.
4. Написан краткий отчет `005.1-fix-device-api-auth.report.[HHMM].md` в папке с датой.
