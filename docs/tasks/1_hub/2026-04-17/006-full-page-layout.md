# Task 1_hub/006: Полноэкранный Layout и общие UI-компоненты

## 🎯 Цель
Изменить дизайн страниц фронтенда Hub, перейдя от "панельного" вида к полноценному веб-приложению (Full-page Layout). Создать переиспользуемые компоненты навигации (Header, Sidebar, Layout) и вынести их в библиотеку `ui-kit` для будущего использования в других микросервисах.

## 🧭 Контекст
Сейчас страницы выглядят как отдельные карточки/панели. Платформе нужен единый стиль: верхняя панель (Header) с информацией о пользователе/сервисе, боковое меню (Sidebar) для навигации по разделам и основная область контента на всю оставшуюся высоту и ширину экрана. Так как у нас микрофронтенды (или просто несколько React-приложений в монорепе), эти базовые структурные элементы должны быть общими.

## 📋 Задачи для выполнения

### 1. Библиотека `ui-kit` (`libs/ui-kit/src/components/layout`)
- Создать компоненты:
  - `Header`: верхняя панель (логотип/название сервиса, кнопка Logout, профиль).
  - `Sidebar`: боковая панель с ссылками навигации. Должна принимать массив элементов меню (название, иконка, ссылка) через props.
  - `AppLayout`: компонент-обертка, который объединяет `Header`, `Sidebar` и `children` (основной контент). Должен занимать `100vh` и `100vw`, контентная часть должна скроллиться независимо от сайдбара.
- Использовать `react-bootstrap` (Navbar, Nav, Container и т.д.) или кастомные стили (CSS Modules / Styled Components), чтобы сделать верстку адаптивной и полноэкранной.
- Экспортировать эти компоненты из `libs/ui-kit/src/index.ts`.

### 2. Применение Layout в Hub (`apps/hub/frontend`)
- Обновить `App.tsx` (или роутер): обернуть все защищенные страницы (например, `/devices`, `/`) в новый `AppLayout` из `@app/ui-kit`.
- Передать в `Sidebar` актуальные ссылки для Hub (например, "Dashboard" -> `/`, "Devices" -> `/devices`).
- Убрать старые локальные навигационные панели (если они были в Hub).
- Страница логина (`/login`) **НЕ** должна использовать этот Layout (она должна остаться по центру экрана).

### 3. Тестирование и проверка (ОБЯЗАТЕЛЬНО)
- Убедиться, что сборка `ui-kit` и `hub/frontend` проходит успешно (`yarn typecheck`, `yarn lint`).
- **ОБЯЗАТЕЛЬНО:** Использовать навык (skill) "WebApp tester" (или запустить браузер/скриншотер), чтобы визуально проверить:
  1. Layout занимает весь экран без лишних отступов у `<body>`.
  2. Сайдбар корректно отображается слева, Хедер — сверху.
  3. Переход по ссылкам в сайдбаре работает (контент меняется, страница не перезагружается целиком).
  4. Страница логина отображается без сайдбара и хедера.

## 🏁 Критерии приемки (DoD)
1. Компоненты `Header`, `Sidebar` и `AppLayout` созданы в `ui-kit` и экспортируются.
2. Фронтенд Hub использует новый `AppLayout` для защищенных страниц.
3. Верстка занимает весь экран (100vh).
4. Агент провел визуальное/функциональное тестирование с помощью WebApp tester и подтвердил работоспособность.
5. Написан отчет `006-full-page-layout.report.[HHMM].md`.
