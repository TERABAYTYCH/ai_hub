**Доработка #9 - 2026-04-23 — Обновление JWT после сохранения**

После успешного сохранения настроек вызывается `/auth/refresh` и обновляется JWT через `login(refreshData.accessToken)`. Это позволяет сразу увидеть изменения без перезагрузки страницы.

**Файлы:** `apps/hub/frontend/src/pages/MicroservicesSettings.tsx`

**Доработка #10 - 2026-04-23 — Refresh JWT на visibilitychange**

В AuthProvider добавлен event listener на `visibilitychange`. При возврате на вкладку (`document.visibilityState === 'visible'`) вызывается refresh JWT для получения актуальных MICROSERVICES_ACCESS.

Применяется ко всем фронтендам использующим ui-kit AuthProvider (Hub, Pulse, Service).

**Файлы:** `libs/ui-kit/src/providers/AuthProvider.tsx`

**Доработка #11 - 2026-04-23 — Исправлен refresh JWT с user**

При вызове `login(refreshData.accessToken)` не передавался `user`, из-за чего `user` становился `null` и Layout не обновлялся. Теперь передаётся `refreshData.user` из ответа `/auth/refresh`.

**Файлы:** `apps/hub/frontend/src/pages/MicroservicesSettings.tsx`
