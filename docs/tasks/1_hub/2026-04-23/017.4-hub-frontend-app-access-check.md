# Тикет 017.4: Hub Frontend — Проверка доступа в App.tsx

**Дата:** 2026-04-23  
**Домен:** 1_hub  
**Родительский план:** PERMISSIONS_001 (Управление доступом к микросервисам)

---

## Цель

Добавить проверку доступа к микросервисам в `useDynamicRoutesConfig()`. Если сервис заблокирован (`microservicesAccess[serviceId] === false`), редиректить на `/{serviceId}/lock`.

**Важно:** При отсутствии доступа блокируются ВСЕ дочерние роуты `/{serviceId}/*`, не только корневой `/{serviceId}`.

---

## Контекст

Сейчас `useDynamicRoutesConfig()` в `apps/hub/frontend/src/App.tsx` загружает модули из всех сервисов без проверки доступа. При прямом входе на `/service/dashboard` пользователь попадает на remote модуль Service, хотя Service заблокирован.

---

## Требуемое состояние

### Изменить useDynamicRoutesConfig в App.tsx

**Файл:** `apps/hub/frontend/src/App.tsx`

1. Добавить `useAuth` в импорт из `@ject-hub/ui-kit`
2. В `useDynamicRoutesConfig()`:
   - Получить `user` через `useAuth()`
   - Извлечь `microservicesAccess = user?.microservices || {}`
   - Для каждого manifest проверить `isLocked = microservicesAccess[manifest.serviceId] === false`
   - Если `isLocked`:
     - НЕ добавлять navigation routes
     - Добавить wildcard route `/{serviceId}/*` → `/{serviceId}/lock`

```typescript
const useDynamicRoutesConfig = () => {
  const { user } = useAuth();
  const { manifests } = useMicroserviceManifests();
  const microservicesAccess = user?.microservices || {};

  const routes = manifests.flatMap(manifest => {
    const isLocked = microservicesAccess[manifest.serviceId] === false;

    if (isLocked) {
      // Блокируем ВСЕ дочерние роуты сервиса
      return [{
        path: `/${manifest.serviceId}/*`,
        element: <Navigate to={`/${manifest.serviceId}/lock`} replace />,
      }];
    }

    // Доступный сервис — добавляем navigation routes
    return manifest.navigation.map(nav => ({
      path: nav.path,
      element: (
        <ProtectedRoute>
          <Layout>
            <LazyModule serviceId={manifest.serviceId} modulePath={nav.module} />
          </Layout>
        </ProtectedRoute>
      ),
    }));
  });

  return routes;
};
```

**Примечание:** Route `/{serviceId}/*` перехватывает ВСЕ дочерние пути (`/service`, `/service/dashboard`, `/service/settings` и т.д.)
