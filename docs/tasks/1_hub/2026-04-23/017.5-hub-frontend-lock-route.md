# Тикет 017.5: Hub Frontend — /lock route

**Дата:** 2026-04-23  
**Домен:** 1_hub  
**Родительский план:** PERMISSIONS_001 (Управление доступом к микросервисам)

---

## Цель

Добавить routes `/pulse/lock` и `/service/lock` для отображения LockPage при попытке зайти на заблокированный сервис через Hub.

---

## Контекст

В 017.4 добавлен wildcard redirect `/{serviceId}/*` → `/{serviceId}/lock`. Теперь нужно создать эти `/lock` routes в Hub App.tsx.

LockPage показывает "Модуль недоступен" с кнопкой "На главную".

---

## Требуемое состояние

### Добавить /lock routes в App.tsx

**Файл:** `apps/hub/frontend/src/App.tsx`

1. Импортировать `LockPage` из `@ject-hub/ui-kit`
2. Добавить static routes для `/pulse/lock` и `/service/lock`

```typescript
import { LockPage } from '@ject-hub/ui-kit';

// в staticRoutes добавить:
{ path: '/pulse/lock', element: <LockPage serviceName="Pulse" returnUrl="/pulse" /> },
{ path: '/service/lock', element: <LockPage serviceName="Service" returnUrl="/service" /> },
```
