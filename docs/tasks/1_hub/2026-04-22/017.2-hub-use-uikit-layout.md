# 017.2: Hub использует Layout из ui-kit

## Цель

Убедиться, что Hub использует Layout из ui-kit напрямую, без дублирования логики.

## Задачи

### 1. Аудит текущего состояния

Проверить:
- `apps/hub/frontend/src/components/layout/Layout.tsx` — текущий Layout
- `libs/ui-kit/src/components/layout/AppLayout.tsx` — ui-kit Layout
- `libs/ui-kit/src/components/layout/index.ts` — экспорты

### 2. Анализ дублирования

Текущий Hub Layout.tsx:
```typescript
import { AppLayout, MenuItem, useAuth, useMicroserviceManifests } from '@ject-hub/ui-kit';

export function Layout({ children }: { children: ReactNode }) {
  const { user, logout } = useAuth();
  const { manifests } = useMicroserviceManifests();
  // ... logic для построения menuItems ...
  return (
    <AppLayout menuItems={menuItems} serviceName="Ject Hub" username={username} onLogout={logout}>
      {children}
    </AppLayout>
  );
}
```

**Вопрос:** Является ли эта логика дублированием или необходимой адаптацией?

### 3. Варианты решения

**Вариант A:** Hub Layout.tsx остаётся как есть (использует ui-kit AppLayout + добавляет свою логику меню)
- Плюс: Логика menuItems из manifests изолирована в Hub
- Минус: Дублирование структуры файла

**Вариант B:** Вынести логику menuItems в ui-kit
- Создать `ui-kit/src/components/layout/Layout.tsx` который инкапсулирует логику manifests
- Hub просто импортирует `Layout` из ui-kit
