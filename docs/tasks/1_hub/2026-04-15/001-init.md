# Task 100: Инициализация базовой инфраструктуры сервиса Hub

## 🧭 Контекст

Сервис **Hub** является центральным узлом платформы (Port 3000). Он отвечает за аутентификацию пользователей, управление лицензиями и хранение мастер-базы устройств. В рамках этой задачи необходимо заложить базовый каркас бэкенда (NestJS) и фронтенда (React + Vite), а также настроить подключение к изолированной базе данных MySQL.

## 🎯 Цели

1. Развернуть базовое NestJS приложение в `apps/hub/backend`.
2. Развернуть базовое React приложение в `apps/hub/frontend`.
3. Настроить подключение к БД MySQL для бэкенда через TypeORM.
4. Создать первичные DTO и интерфейсы в `libs/contracts` для модуля авторизации.
5. Подготовить Docker-окружение для локального запуска сервиса Hub.

## 🛠 Технические требования

- **Backend:** NestJS, строгий TypeScript, TypeORM (без `synchronize: true`, только миграции).
- **Frontend:** React 18+, Vite, маршрутизация (React Router).
- **Shared:** Все DTO выносятся в `libs/contracts/hub`.
- **Инфраструктура:** Обновление корневого `docker-compose.yml` для добавления контейнеров `hub-backend`, `hub-frontend` и `hub-db` (MySQL).

---

## 📋 План выполнения (Step-by-Step)

### Шаг 1: Настройка общих контрактов (Shared Contracts)

- Создать директорию `libs/contracts/hub/auth`.
- Описать базовые интерфейсы: `LoginRequestDto`, `LoginResponseDto`, `UserJwtPayload`.
- Экспортировать их через `index.ts` библиотеки контрактов.

### Шаг 2: Инициализация Backend (NestJS)

- Сгенерировать базовую структуру NestJS в `apps/hub/backend`.
- Настроить `app.module.ts` с подключением `ConfigModule` и `TypeOrmModule`.
- Создать `AuthModule` (пока без сложной логики, только каркас контроллера и сервиса).
- Настроить скрипты в `package.json` для локальной разработки.

### Шаг 3: Инициализация Frontend (React + Vite)

- Сгенерировать базовое Vite-приложение (React + TS) в `apps/hub/frontend`.
- Настроить `vite.config.ts` (порт 3000).
- Создать базовую структуру папок (`src/pages`, `src/components`, `src/store`).
- Добавить простую страницу Login, которая будет использовать контракты из `libs/contracts`.

### Шаг 4: Инфраструктура (Docker & БД)

- Добавить в корневой `docker-compose.yml` сервисы для Hub:
  - `hub-db`: MySQL 8.0 (порт 3306, проброс на 33061 для локальной отладки).
  - `hub-backend`: Node.js контейнер (порт 3000).
  - `hub-frontend`: Node.js контейнер для Vite dev-сервера.
- Убедиться, что сервисы корректно стартуют и видят друг друга во внутренней сети Docker.

## ✅ Ожидаемый результат

По завершении задачи сервис Hub должен успешно запускаться через Docker Compose, бэкенд должен подключаться к своей базе данных, а фронтенд — отдавать базовую страницу на порту 3000. Все общие типы должны импортироваться из `libs/contracts`.

После выполнения необходимо сгенерировать отчет `100-hub.report.md`.
